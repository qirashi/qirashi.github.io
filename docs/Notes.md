# Тестовый файл со всеми типами Admonitions

Вот демонстрация всех специальных блоков разметки в GitHub Flavored Markdown:

## Основные типы

> [!NOTE]
> Это нейтральное примечание с полезной дополнительной информацией. Используется для пояснений, которые не являются критичными, но помогают лучше понять контекст.

> [!TIP]
> Вот полезный совет! Попробуйте использовать горячие клавиши `Ctrl+S` для быстрого сохранения файлов. Это ускорит вашу работу.

> [!IMPORTANT]
> Обязательно сделайте бэкап базы данных перед обновлением! Это критически важное действие для предотвращения потери данных.

## Предупреждения и опасности

> [!WARNING]
> При удалении этой папки все файлы будут безвозвратно утеряны. Продолжайте с осторожностью.

> [!CAUTION]
> Вы собираетесь выполнить команду `rm -rf /`, которая может необратимо повредить систему! Не выполняйте эту команду, если не уверены на 100%.

## Информационные блоки

> [!ABSTRACT]
> В этой статье рассматриваются все типы специальной разметки в GitHub, их назначение и примеры использования для улучшения документации.

> [!SUMMARY]
> Admonitions - это специальные блоки в GFM для выделения важной информации, предупреждений, советов и других типов контента.

> [!INFO]
> GitHub Flavored Markdown был расширен поддержкой этих блоков в 2023 году. Ранее для similar functionality использовались кастомные решения.

## Статусные блоки

> [!SUCCESS]
> Операция завершена успешно! Все файлы были загружены на сервер без ошибок.

> [!CHECK]
- [x] Задача выполнена
- [x] Код протестирован  
- [x] Документация обновлена

> [!DONE]
> Процесс миграции данных завершен. Система готова к использованию.

## Блоки вопросов и помощи

> [!QUESTION]
> Как часто следует обновлять зависимости в проекте? Рекомендуется делать это ежеквартально или при обнаружении уязвимостей.

> [!HELP]
> Если у вас возникли проблемы с установкой, проверьте версию Node.js (должна быть не ниже 18.0) и убедитесь, что у вас есть права на запись в целевую директорию.

> [!FAQ]
> **В: Поддерживаются ли эти блоки в GitLab?**
> 
> **О:** На момент написания GitLab не имеет нативной поддержки этих admonitions, но есть плагины и кастомные решения.

## Блоки ошибок и проблем

> [!FAILURE]
> Ошибка компиляции: не удалось разрешить зависимость `com.example:library:1.0.0`. Проверьте настройки репозитория.

> [!FAIL]
> Тест `testUserRegistration` не пройден. Время ответа сервера превысило лимит в 2000 мс.

> [!MISSING]
> В конфигурационном файле отсутствует обязательный параметр `api_key`. Добавьте его в секцию `[auth]`.

> [!DANGER]
> Обнаружена критическая уязвимость CVE-2024-12345 в используемой версии библиотеки! Немедленно обновите до версии 2.1.0 или выше.

> [!ERROR]
> Системная ошибка: невозможно подключиться к базе данных. Проверьте настройки подключения и убедитесь, что сервер БД запущен.

> [!BUG]
> **Описание бага:** При скроллинге на мобильных устройствах меню навигации иногда исчезает.
> 
> **Шаги воспроизведения:**
> 1. Откройте сайт на мобильном устройстве
> 2. Прокрутите страницу вниз
> 3. Быстро прокрутите вверх
> 
> **Ожидаемый результат:** Меню остается видимым
> **Фактический результат:** Меню исчезает

## Примеры и цитаты

> [!EXAMPLE]
> Вот пример использования функции:
> ```javascript
> // Создание admonition
> function createAdmonition(type, content) {
>   return `> [!${type.toUpperCase()}]\n> ${content}`;
> }
> 
> // Использование
> const note = createAdmonition('NOTE', 'Это пример');
> console.log(note);
> ```

## Кастомные заголовки

Вы также можете использовать кастомные заголовки:

> [!CUSTOM|label:МойЗаголовок]
> Это блок с кастомным заголовком. В GitHub он будет отображаться как `[!CUSTOM]`, но некоторые другие реализации admonitions позволяют изменять заголовок.

---

**Примечание:** Все эти блоки поддерживаются в GitHub Flavored Markdown и будут корректно отображаться на GitHub.com, в репозиториях и файлах `.md`.
